# Парсер товаров с pearlpool.ru для WooCommerce

PHP-скрипт для автоматического парсинга товаров с сайта pearlpool.ru и отправки их в WooCommerce через REST API.

## Требования

- PHP 7.4 или выше
- Расширение cURL
- Расширение DOM

## Установка

1. Скопируйте файл `pearlpool_parser.php` на ваш сервер
2. Отредактируйте конфигурацию в начале файла при необходимости:

```php
$config = [
    'api_url' => 'https://9677018686.myjino.ru/basik/wp-json/wc/v3',
    'consumer_key' => 'ck_351a30e18d90c3db83151bd7eda51c971eb7d510',
    'consumer_secret' => 'cs_8ffea5d680b70c3b38123ef59071367bc1ba28a8',
];

$parser_config = [
    'base_url' => 'https://pearlpool.ru/',
    'catalog_url' => 'https://pearlpool.ru/catalog/',
    'delay_between_requests' => 1, // задержка между запросами в секундах
    'max_products_per_run' => 50, // максимум товаров за один запуск
];
```

## Запуск вручную

```bash
php /path/to/pearlpool_parser.php
```

## Настройка Cron

Для автоматического запуска каждые 15 минут добавьте в crontab:

```bash
*/15 * * * * /usr/bin/php /path/to/pearlpool_parser.php >> /path/to/parser.log 2>&1
```

### Примеры расписаний:

- Каждый час: `0 * * * *`
- Каждые 30 минут: `*/30 * * * *`
- Раз в день в 3:00: `0 3 * * *`
- Каждые 15 минут: `*/15 * * * *`

## Функционал

1. **Автоматическое определение категорий** - скрипт сканирует каталог и находит все категории товаров
2. **Парсинг товаров** - извлекает название, цену, описание, изображения, SKU
3. **Синхронизация с WooCommerce**:
   - Создание новых товаров
   - Обновление существующих (по SKU или названию)
   - Загрузка изображений
4. **Защита от перегрузки**:
   - Лимит товаров за один запуск
   - Задержки между запросами
5. **Логирование** - все действия записываются в stdout (перенаправляйте в лог-файл)

## Структура данных товара

Скрипт извлекает следующие данные:
- Название товара
- Описание
- Краткое описание
- Цена
- Изображения (до 5 шт)
- Артикул (SKU)
- Категория

## Решение проблем

### Товары не создаются
- Проверьте правильность API ключей WooCommerce
- Убедитесь, что у ключей есть права на запись
- Проверьте логи ошибок

### Медленная работа
- Увеличьте `max_products_per_run` для большего количества товаров за запуск
- Уменьшите `delay_between_requests` (но не ставьте 0)

### Ошибки SSL
- Скрипт уже отключает проверку SSL (`CURLOPT_SSL_VERIFYPEER => false`)
- Для продакшена рекомендуется настроить правильные SSL сертификаты

## Безопасность
